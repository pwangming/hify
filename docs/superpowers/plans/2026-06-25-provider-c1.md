# provider C1（读侧：模型校验 Facade + 成员侧模型列表）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现 provider C1——provider 首个对外门面 `ProviderFacade.findUsableChatModel` + 成员侧「列可用 chat 模型」接口；app 接入做 model_id 校验；前端弹窗加模型选择器。前后端一体、一起验证。配套 spec：`docs/superpowers/specs/2026-06-25-provider-c1-design.md`。

**Architecture:** provider 新增只读 `ModelQueryService`（两次 MyBatis-Plus wrapper 查询拼「可用」判定，不写 SQL）；`ProviderFacade`(api) + `ProviderFacadeImpl`(service) 暴露给 app；`ModelQueryController` 暴露成员族 `GET /api/v1/provider/models`。app 的 `AppService` 注入 `ProviderFacade`，create/update 校验非空 model_id。前端 app 弹窗加 `el-select` 模型选择器。

**Tech Stack:** Spring Boot 3 / Java 21 / MyBatis-Plus / Spring Modulith；Vue 3 + TS + Element Plus + vitest。

## Global Constraints

- 后端连库测试（Testcontainers）本轮仍推迟，全部 Mockito mock mapper / mock Facade。
- 判定测试结果**不 grep `BUILD SUCCESS`**（`-q` 会静音）——看测试计数与进程退出码。
- Long 一律序列化为 JSON string：`ModelView.id`。
- **不写手写 SQL**：可用性判定用 `selectById` / `selectList` / `selectBatchIds` + LambdaQueryWrapper 拼装（spec §2.2）。
- **不改任何 package-info**（app 白名单已含 `provider::api`，provider/api 已 `@NamedInterface`）；**无 Flyway 迁移**（不动表）。
- 错误码：app 段新增 `16002`（MODEL_NOT_USABLE）；provider 段本轮不新增。
- 每完成一个 Task，向 `docs/self-check.md` 追加一条自检（沿用既有格式：本步做了什么 / 怎么自证 / 已知遗留），与该 Task 提交同批 commit。
- 提交粒度：每个 Task 末尾 commit 一次；先建分支 `feat/provider-c1`（从 main 切）。

**关键决策（落位）：**
- `ModelView` 放 `provider/api/dto/`（首个对外 DTO，被 Facade 签名与前端对齐引用）。
- `ProviderFacade` 放 `api/`、`ProviderFacadeImpl` 放 `service/`（Modulith 强制：Facade 是唯一接口抽象）。
- 「可用」= 模型 enabled + type=chat + 供应商 enabled（连带，spec §决策3）。
- model_id 选填：null 放行、非空才校（spec §决策2）。

### Task 1: ModelView DTO + ModelQueryService（可用性查询）

**Files:**
- Create: `server/src/main/java/com/hify/provider/api/dto/ModelView.java`
- Create: `server/src/main/java/com/hify/provider/service/ModelQueryService.java`
- Test: `server/src/test/java/com/hify/provider/service/ModelQueryServiceTest.java`

**Interfaces:**
- `record ModelView(Long id, String name, String type, String providerName)`
- `ModelQueryService`（@Service，无 @Transactional）注入 `AiModelMapper` + `ModelProviderMapper`：
  - `Optional<ModelView> findUsableChatModel(Long modelId)`
  - `List<ModelView> listUsableChatModels(String type)`（type 为空按 "chat"）

- [ ] **Step 1: 写失败测试**（mock 两个 mapper）。用例：
  - `findUsableChatModel` 正常：模型 enabled+chat、供应商 enabled → 返回 ModelView，`name`/`providerName`/`type=chat` 正确。
  - 模型不存在（mapper 返回 null）→ `Optional.empty()`。
  - 模型 `status=disabled` → empty。
  - 模型 `type=embedding` → empty（且不应去查供应商）。
  - 供应商 `status=disabled` → empty（连带校验）。
  - `listUsableChatModels`：给两个 chat 模型分属一个 enabled、一个 disabled 供应商 → 只回 enabled 那条；providerName 映射正确。
  - `listUsableChatModels` 无 enabled chat 模型 → 返回空 `List`（非 null）。
  > 构造 `AiModel`/`ModelProvider` 实体作桩；注意 `@TableLogic` 由真实查询加 `deleted=false`，单测里 mock 直接给已过滤的返回即可。

- [ ] **Step 2: 运行确认失败** `cd server && mvn -q -Dtest=ModelQueryServiceTest test`，Expected：编译失败（类未定义）。

- [ ] **Step 3: 写最小实现**（spec §2.2 两查算法）：
  - `findUsableChatModel`：`selectById(modelId)` 守 null/disabled/非chat → `selectById(providerId)` 守 null/disabled → 拼 ModelView。
  - `listUsableChatModels`：`selectList(eq type, eq status='enabled')` → 收集 distinct providerId → 空则返回 `List.of()` → `selectBatchIds` 过滤 enabled 建 `id→name` 映射 → 过滤+映射，按 name 排序。
  - 用 `ProviderStatus.ENABLED.value()` / `ModelType.CHAT.value()` 常量，不硬编码字符串。

- [ ] **Step 4: 运行确认通过**（同上命令绿）。

- [ ] **Step 5: 自检 + commit**（`docs/self-check.md` 追加；`feat(provider): C1 模型可用性只读查询 ModelQueryService + ModelView`）。

**Interfaces:**
- `AppError.MODEL_NOT_USABLE(16002, BAD_REQUEST, "所选模型不存在或不可用")`
- `AppService` 构造增 `ProviderFacade`；私有 `assertModelUsableIfPresent(Long modelId)`：非空时 `findUsableChatModel` empty → 抛 `BizException(AppError.MODEL_NOT_USABLE)`。

- [ ] **Step 1: 改/写失败测试**：
  - create：`modelId` 非空 + Facade 返回非空 → 落库成功（verify insert）。
  - create：`modelId` 非空 + Facade 返回 empty → `BizException` code=16002。
  - create：`modelId == null` → **不调** Facade（`verifyNoInteractions` 或 `never()`）、正常落库。
  - update：同 create 两类（非空可用/不可用、null 放行）。
  - **既有用例**：给所有构造 `AppService` 的地方补 `ProviderFacade` mock，默认桩 `findUsableChatModel(any)` 返回非空（避免既有 create/update 测试因 NPE 红）。
- [ ] **Step 2: 运行确认失败** `mvn -q -Dtest=AppServiceTest test`（16002 未定义 / 校验未接）。
- [ ] **Step 3: 写实现**：AppError 加枚举值；AppService 构造注入 Facade；create/update 在 `appMapper.insert/updateById` 前调 `assertModelUsableIfPresent`。
- [ ] **Step 4: 确认通过**；再跑全量 `mvn test` 确认 Modulith/ArchUnit 仍绿（app 首次 import provider::api，边界已允许）。
